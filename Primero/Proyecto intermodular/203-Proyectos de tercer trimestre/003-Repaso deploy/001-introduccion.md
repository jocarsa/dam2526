Nosotros desarrollamos en nuestro equipo de desarrollo
Es un equipo local
Con un OS con ventanas
Es muy potente
Es muy cómodo
Está en nuestra casa

Pero luego la aplicación tiene que estar alojada en un servidor
Es un equipo remoto
Es un OS de consola (generalmente Ubuntu Server)
Es poco potente
No es cómo de usar
No está en nuestra casa

Deploy fácil consiste en conectar mediante FTP y transferir los archivos mediante FTP
Deploy más complejo pero mejor consiste en conectar mediante Github
Desarrollo push a github, servidor hace pull desde github

Para proyectos grandes, con versionado, etc etc recomiendo GitHub
Para proyectitos recomiendo FTP

Recordamos que:
Tenemos un servidor contratado (en vuestro caso puede ser un servidor maquina virtual)
Con un dominio y un certificado SSL

PAra conectar mediante FTP al servidor recomiendo Filezilla
Si usamos apache la carpeta de publicación es /var/www/html/XXXXX
